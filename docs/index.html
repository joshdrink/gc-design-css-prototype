Welcome to the CSS Proposal

Below, you'll find examples of two components (blockquotes, buttons) and some typography.

Each component has its own structural CSS found in its folder. This structural style refers to any paint related style using CSS variables. The typography is styled using CSS variables as well, but its CSS can be found in the root CSS folder as it refers to the product as a whole.

CSS variables in this proposal are written like this: background-color: var(--theme-brand-color-1, var(--default-brand-color-1));, where the default brand color is the repository's default theme, and the theme brand color refers to a variable that the user of the design system can control if they'd like to alter things.

Inside ./app/css/ you'll find a file called variables-theme.css with a handful of variables that have been commented out. This file emulates a user of the design system building their own theme for their brand. Uncomment the variables and save the file with the node command running to see how altering these variables provide a themeing experience.
